## 1. 产品概述

### 1.1 产品愿景

将 AI Edge Gallery 从一个**模型推理工具**扩展为一个**设备端 AI 自动化平台**，让用户能够通过可视化编排 LLM 与设备能力，创建个性化的 AI 工作流，实现类似苹果快捷指令的自动化体验。

### 1.2 核心价值主张

- **隐私优先**：所有推理和处理均在设备端完成，数据不离开设备
- **离线可用**：无需网络，自动化随时随地运行
- **LLM 驱动的智能**：利用大语言模型理解用户意图、决策分支、生成内容
- **高度可扩展**：支持用户自定义 Actions 和 Skills

### 1.3 目标用户

| 用户类型 | 使用场景 |
|---------|---------|
| 效率爱好者 | 创建日常任务自动化（如整理照片、发送模板消息） |
| 开发者 | 构建 AI 增强的工作流，测试 LLM 能力 |
| 隐私敏感用户 | 希望数据留在本地，不依赖云服务 |
| 内容创作者 | 利用 LLM 多模态能力处理图片、音频 |

## 2. 产品定位与范围

### 2.1 与苹果快捷指令的对比

| 维度 | Apple Shortcuts | AI Edge Gallery Workflow (目标) |
|------|-----------------|-------------------------------|
| **核心理念** | 设备操作自动化 | AI + 设备操作智能化 |
| **智能能力** | 基础条件判断 | LLM 推理、意图理解、内容生成 |
| **多模态** | 有限 | 强大（图像、音频、文本联合处理） |
| **隐私** | 本地优先 | 100% 设备端 |
| **目标用户** | 普通用户 | 科技爱好者、开发者 |

### 2.2 差异化优势

1. **AI Native**：每个工作流都可以调用 LLM 进行理解、决策、生成
2. **多模态原生**：原生支持图像、音频、文本的联合处理
3. **开源透明**：基于开源模型和框架，无供应商锁定

### 4.5 控制流

#### 4.5.1 条件分支

```
IF [条件表达式]
    → Action A
ELSE IF [条件表达式]
    → Action B
ELSE
    → Action C
END IF
```

**支持的条件操作符：**

| 操作符 | 示例 | 说明 |
|--------|------|------|
| **等于** | var == "value" | 字符串相等 |
| **不等于** | var != "value" | 字符串不等 |
| **包含** | var contains "text" | 包含子串 |
| **大于/小于** | num > 10 | 数值比较 |
| **为空** | var is empty | 检查空值 |
| **LLM 判断** | LLM 判断 [条件] | LLM 理解条件 |

#### 4.5.2 循环结构

```
REPEAT [N] times
    → Action
END REPEAT

FOR EACH [item] IN [list]
    → Action(item)
END FOR EACH

WHILE [condition]
    → Action
END WHILE
```

### 4.6 工作流执行引擎

#### 4.6.1 架构设计

```

│                                                                  │

```

#### 4.6.2 执行流程

```
1. Trigger 触发 (用户点击 / 定时器到期 / 事件发生)
   ↓
2. Executor 加载工作流定义
   ↓
3. 初始化变量存储
   ↓
4. 按顺序执行 Actions:
   a. 从 Registry 获取 Action 实例
   b. 绑定输入变量
   c. 执行 Action
   d. 保存输出到变量
   e. 处理控制流 (条件/循环)
   ↓
5. 工作流完成
   ↓
6. 显示结果 / 发送通知
```

## 5. 非功能需求

### 5.1 性能

| 指标 | 目标 |
|------|------|
| 工作流启动时间 | < 500ms |
| Action 执行延迟 | 与直接调用一致 |
| 工作流保存/加载 | < 100ms |
| UI 响应性 | 60 FPS |

### 5.2 隐私与安全

- 所有 LLM 推理完全在设备端完成
- 不收集用户工作流数据
- 工作流文件存储在应用私有目录
- 敏感变量（如 API Key）加密存储

### 5.3 可用性

- 支持深色/浅色主题
- 支持多语言（英文为主，中文待定）
- 辅助功能：屏幕阅读器支持
- 错误处理：优雅降级，清晰错误提示

### 5.4 兼容性

| 平台 | 最低版本 |
|------|---------|
| Android | 12 (API 31) |
| iOS | 17 |

| 用户留存 | 30日留存 +20% |
